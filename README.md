# Bot Factory — лендинг-сайт студии

Одностраничный сайт студии **Bot Factory** (Минск, Беларусь). Студия делает сайты, Telegram-боты, Mini Apps и AI-интеграции под ключ. Лендинг рассказывает об услугах, процессе, ценах и собирает заявки в Telegram.

## Кратко

- **Тип:** статический сайт, без сборщика и npm-зависимостей.
- **Стек:** HTML5 + CSS3 + ванильный JavaScript (ES2020+).
- **Анимации скролла:** [Lenis 1.0.42](https://github.com/darkroomengineering/lenis) — лежит локально в `assets/vendor/`, без CDN.
- **Шрифты:** Google Fonts — `Manrope` (основной) и `JetBrains Mono` (моно/лейблы).
- **Язык интерфейса:** русский (`<html lang="ru">`).
- **Целевая аудитория:** бизнес в Беларуси (Минск).

## Секции лендинга

1. **Прелоадер** — анимированный «робот» + полоса прогресса.
2. **Шапка** — логотип, десктопная навигация, CTA «Обсудить проект», бургер-меню для мобильных.
3. **Hero** — терминал-блок с морфингом фразы (`приносят клиентов` / `работают 24/7` / `продают`).
4. **Proof** — три карточки: НПД и УНП, договор и гарантия, концепт до оплаты.
5. **Услуги (`#services`)** — 4 строки с превью справа: Сайты, Telegram-боты, Mini Apps, AI-интеграции.
6. **Offer** — бесплатный концепт за 48 часов.
7. **Интеграции** — две бегущие строки (amoCRM, Bitrix24, 1С, Yclients, OpenAI, ERIP и т. д.).
8. **Процесс (`#process`)** — таймлайн из 5 шагов: Бриф → Концепт → Разработка → Запуск → Поддержка.
9. **Цены (`#pricing`)** — табы (Сайты / Telegram-боты / AI / интеграции), три тарифа в каждом, контент рендерится из `pricingContent` в `script.js`.
10. **Гарантии** — договор, гарантия 2 месяца, сроки, официальное оформление.
11. **Final CTA (`#contact`)** — кнопки в Telegram, email, телефон.
12. **FAQ** — `<details>`-аккордеон.
13. **Подвал** — контакты, услуги, соцсети, юридическая строка.

## Поведение JS

Файл [script.js](script.js):

- **Прелоадер.** Прогресс инкрементируется по таймеру; уважает `prefers-reduced-motion`.
- **Lenis.** Включается, только если `window.Lenis` загрузился и пользователь не запросил уменьшение анимаций.
- **Плавный скролл по якорям.** Работает с любым `a[href^="#"]`, корректно обновляет `history`. Если в URL уже есть `#hash`, докручивает к секции после загрузки.
- **Шапка.** Класс `is-scrolled` при `scrollY > 100`.
- **Мобильное меню.** Управляет `inert`, `aria-hidden`, `aria-expanded`, ловит `Escape`, возвращает фокус на кнопку. Закрывает меню по клику на пункт.
- **Hero-морфинг.** WAAPI-анимация смены фразы каждые 2.6 с.
- **`IntersectionObserver`.** На все `.section-reveal` навешивает класс `is-visible` при появлении в вьюпорте.
- **Услуги.** Превью справа перерисовывается при `mouseenter` / `focus` / `click` по строке.
- **Цены.** Табы поддерживают клавиатуру (`ArrowLeft/Right`, `Home`, `End`, `Enter`, `Space`), `aria-selected`, `aria-controls`, `tabindex`, `aria-live`.
- **Кастомный курсор.** Точка + кольцо с лагом, активируется на `[data-cursor], a, button`. На устройствах без указателя курсор скрывается через CSS.

## Кеш-бастинг

Стили и скрипт подключаются с query-версией, чтобы CDN/браузеры не отдавали старое:

```html
<link rel="stylesheet" href="styles.css?v=20260505-lenis-css" />
<script src="script.js?v=20260505-mobile-a11y-menu"></script>
```

При значимых правках `styles.css` или `script.js` обновляйте суффикс `?v=...` в [index.html](index.html).

## Запуск локально

Любой статический сервер из корня проекта. Примеры:

```bash
# Python 3
python -m http.server 8000

# Node
npx http-server -p 8000 -c-1

# VS Code: расширение Live Server
```

Открыть `http://localhost:8000/`.

> Прямое открытие `index.html` через `file://` тоже работает, но шрифты Google и относительные пути ведут себя предсказуемее на http-сервере.

## Доступность

- `prefers-reduced-motion` уважается: прелоадер ускоряется, плавный скролл и Lenis отключаются, табы переключаются без `smooth scroll`.
- Мобильное меню корректно работает с фокусом и `inert`.
- Табы цен реализованы как `role="tablist"` с правильным управлением `tabindex` и стрелками.
- FAQ — нативные `<details>` (работают без JS).

## Контент-данные в JS

В [script.js](script.js) есть два словаря, которые управляют контентом:

- `serviceContent` — заголовок, описание и теги для каждой услуги (ровно 4 элемента, индексы привязаны к `data-service` в HTML).
- `pricingContent` — три категории (`sites`, `bots`, `systems`), у каждой `note` и три плана (`badge`, `title`, `price`, `description`, `includes[]`). Количество карточек в HTML (`[data-plan-card="0..2"]`) должно совпадать с длиной `plans`.
